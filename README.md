# CS-230-The-Gaming-Room

The client, The Gaming Room, owner of the popular Android game Draw It or Lose It, wants to expand their game to different operating systems via a cloud provider. 

When creating this Software Design Document, I think I did particularly well discussing the development requirements, server-side, client-side, and development tools, as I was able to highlight their advantages and use sources to do so. 

The most helpful part of the Software Design Document was the UML diagram, as I was able to reference it back and forth when fleshing out the classes that inherit from Entity.

If I were to revise one part of the document, it would be when describing the Domain Model. This is because now that I have worked with the code, I can see some things I didn’t see in the original plan, and I feel they would be valued in the Domain Model section. 

Considering the user’s needs is extremely important because, in the end, they will be the primary users of the application. Their opinion matters the most because the application was made for them. They are the primary actor of the system. For example, when choosing a cloud provider, I was looking for ones that provide exceptional performance for gaming applications, as that is what will be deployed onto the server, and gamers need quick response times in order to play the game correctly. 

I now see that creating a Software Design Document is an essential part of creating an application. This is because it is basically the blueprint for the entire project, its purpose, and the details behind each decision made regarding the construction and idea of the project. In the future, I will definitely be creating a UML class diagram, a chart comparing the pros and cons of each decision, and an executive summary of the project.
